# 📐 Requisitos Não Funcionais (RNF)

Os requisitos não funcionais definem os critérios de operação e as restrições do sistema, garantindo sua qualidade e eficiência.

---

| ID | Categoria | Descrição |
| :--- | :--- | :--- |
| **RNF01** | Confiabilidade | O sistema deve garantir a persistência e integridade dos dados de público e sessões. |
| **RNF02** | Evolução | O sistema deve ser desenvolvido com uma arquitetura que facilite a inclusão de novas unidades ou funcionalidades futuras. |
| **RNF03** | Usabilidade | A interface deve ser intuitiva, permitindo que os funcionários realizem o registro de público com o mínimo de cliques. |
| **RNF04** | Desempenho | As consultas de totalização de público devem ser processadas em tempo real ou em poucos segundos. |
| **RNF05** | Disponibilidade | O sistema deve estar disponível durante todo o horário de operação dos cinemas. |
| **RNF06** | Segurança | Todas as senhas devem ser armazenadas utilizando algoritmos de criptografia (hashing) seguros (ex: BCrypt). |
